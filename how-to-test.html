Thunder Client REST Guide

API Manual Testing Guide

Step-by-step HTTP requests to verify Authentication, Workspaces, Projects, Issues, Comments, and Scoped Permissions.

⚡ Thunder Client Cookie Handling: Because we are using session-based authentication, Thunder Client will automatically capture the sid cookie set by the server on a successful Register or Login request. Subsequent requests made in the client will automatically include this cookie!

1. Authentication Flow

POST /auth/register
201 Created

Create a new user account. This starts a session automatically.

Request Body (JSON)
{
  "name": "Alex Developer",
  "email": "alex@example.com",
  "password": "SecurePassword123"
}
GET /auth/me
200 OK

Fetch the profile details of the currently logged-in user.

Expected Response
{
  "data": {
    "id": "cuid-string",
    "name": "Alex Developer",
    "email": "alex@example.com",
    "createdAt": "date-string",
    "updatedAt": "date-string"
  }
}
POST /auth/logout
204 No Content

Invalidates the session in the database and clears the cookie.

POST /auth/login
200 OK

Authenticate with credentials to open a new session.

Request Body (JSON)
{
  "email": "alex@example.com",
  "password": "SecurePassword123"
}

2. Workspace Management

POST /workspaces
201 Created

Create a new workspace. The creating user is automatically joined to it as an admin.

Request Body (JSON)
{
  "name": "Development Labs"
}
GET /workspaces
200 OK

List all workspaces where the current user is a member.

GET /workspaces/:workspaceId
200 OK

Get detailed information about a workspace. Replace :workspaceId with the ID returned by the creation response.

PATCH /workspaces/:workspaceId
200 OK

Update workspace settings. Allowed for admins only.

Request Body (JSON)
{
  "name": "Design Labs & Engineering"
}

3. Projects Scoped Inside Workspace

POST /workspaces/:workspaceId/projects
201 Created

Create a project inside the workspace. Allowed for admins only.

Request Body (JSON)
{
  "name": "API Service v1",
  "description": "Core Express REST application"
}
GET /workspaces/:workspaceId/projects
200 OK

List projects within this workspace. Supports query sorting/pagination.

PATCH /workspaces/:workspaceId/projects/:projectId
200 OK

Update project attributes. Allowed for admins only.

Request Body (JSON)
{
  "name": "Monolith API Service v1",
  "status": "active"
}
DELETE /workspaces/:workspaceId/projects/:projectId
204 No Content

Soft-delete (archives) a project. Allowed for admins only.

4. Issues Scoped Inside Projects & Workspaces

POST /workspaces/:workspaceId/projects/:projectId/issues
201 Created

Create an issue inside a project. Allowed for admins and members only (viewers blocked).

Request Body (JSON)
{
  "title": "Set up database connection pool",
  "description": "Configure pg.Pool and Prisma connection limits",
  "priority": "high"
}
GET /workspaces/:workspaceId/issues
200 OK

List all issues inside the workspace. Supports filters and paginations.

Example Query String
?page=1&limit=10&sortBy=createdAt&sortOrder=desc&status=todo
GET /workspaces/:workspaceId/projects/:projectId/issues
200 OK

List issues belonging strictly to one project. Supports query limits/sorting.

PATCH /workspaces/:workspaceId/issues/:issueId
200 OK

Update issue fields or perform assignments. Allowed for admins and members only.

Request Body (JSON)
{
  "status": "in_progress",
  "assigneeUserId": "some-member-cuid"
}

5. Comments Scoped Inside Issues

POST /workspaces/:workspaceId/issues/:issueId/comments
201 Created

Post a comment on a target issue. Allowed for admins and members only.

Request Body (JSON)
{
  "body": "PR raised for review. Verified connection parameters locally."
}
GET /workspaces/:workspaceId/issues/:issueId/comments
200 OK

List discussion logs for an issue in chronological order. Authors are attached.

6. Security & Authorization Boundary Checks

GET /workspaces/ws_acme/projects
403 Forbidden

Test: Cross-tenant data isolation.
Log in as your new user (e.g. alex@example.com). Make a query to list projects in the seeded ws_acme workspace (which belongs to Alice/Bob/Carol, not you).

Expected Error Response
{
  "error": {
    "code": "FORBIDDEN",
    "message": "You are not a member of this workspace"
  }
}
POST /workspaces/:workspaceId/projects
403 Forbidden

Test: Role privileges.
Log in as a user who is a member of the workspace but not an admin (e.g., bob@example.com, after updating their password in the database seed). Try to create a project.

Expected Error Response
{
  "error": {
    "code": "FORBIDDEN",
    "message": "You do not have permission to perform this action"
  }
}
PATCH /workspaces/:workspaceId/issues/:issueId
400 Bad Request

Test: Relational Boundary checking.
Log in as an admin of the workspace. Attempt to update/assign an issue to a user ID that does not belong to this workspace membership.

Expected Error Response
{
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "Assignee must be a member of the workspace"
  }
}
